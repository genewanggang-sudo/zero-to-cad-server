# Zero-to-CAD 推理参数

Zero-to-CAD 的推理链路是：8 张 CAD 多视角渲染图 -> Qwen3-VL 微调模型 ->
CadQuery Python 代码 -> STEP/STL。先用默认参数跑通官方样本，再根据效果和资源占用调参。

## 推荐默认值

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `torch_dtype` | `auto` | 自动选择模型推理精度 |
| `device_map` | `auto` | 自动把模型放到可用设备 |
| `max_new_tokens` | `4096` | 限制生成的 CadQuery 代码长度 |
| `do_sample` | `false` | 使用更稳定的确定性生成 |

## 参数说明

| 参数 | 可选值 | 控制内容 | 使用建议 |
| --- | --- | --- | --- |
| `torch_dtype` | `auto`, `bfloat16`, `bf16`, `float16`, `fp16`, `float32`, `fp32` | 模型推理精度，影响显存、速度和稳定性 | 优先用 `auto`；显存不足时再尝试 `bfloat16` 或 `float16` |
| `device_map` | `auto`, `cuda`, `cpu` | 模型运行设备 | 优先用 `auto`；`cpu` 通常很慢，只适合排查问题 |
| `max_new_tokens` | 正整数 | 最多生成多少 token，也就是 CadQuery 代码最长多长 | 默认 `4096`；代码被截断时再增大 |
| `do_sample` | `true`, `false` | 是否启用随机采样 | 默认 `false`；想生成多个候选时再开启 |
| `temperature` | 浮点数 | 随机程度，通常只在 `do_sample=true` 时生效 | 可从 `0.2` 开始试；不建议高于 `0.7` |
| `top_p` | `0-1` 浮点数 | 采样范围，值越低越保守 | 常见尝试值 `0.9`，需配合 `do_sample=true` |
| `top_k` | 正整数 | 每一步只从概率最高的 K 个候选中采样 | 不是优先参数，通常先不调 |
| `num_beams` | 正整数 | Beam search 候选数量 | 默认 `1`；大于 `1` 会更慢、更占显存，是否提升效果需要实测 |
| `repetition_penalty` | 浮点数 | 重复惩罚，减少重复代码 | 默认 `1.0`；重复严重时可试 `1.05-1.1` |

## 调参顺序

1. 先保持默认值，确认官方样本能完成生成和导出。
2. 如果生成代码被截断，优先增大 `max_new_tokens`。
3. 如果显存不足，再调整 `torch_dtype`，优先尝试 `bfloat16` 或 `float16`。
4. 如果想比较多个候选结果，再开启 `do_sample=true`，并从 `temperature=0.2`、`top_p=0.9` 开始。
5. 不要一开始就提高随机性。CAD 代码生成更依赖稳定性和可执行性。

## 输入要求

| 要求 | 说明 |
| --- | --- |
| 图片数量 | 8 张 |
| 视角 | 4 张 front views + 4 张 rear views |
| 内容 | 同一个物体 |
| 尺寸 | 推荐 256x256 |
| 风格 | 干净 CAD 渲染图 |
| 不推荐 | 真实照片、复杂背景、复杂多零件装配 |
